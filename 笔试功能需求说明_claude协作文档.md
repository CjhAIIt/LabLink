# 实验室管理系统新增“笔试中心”功能需求文档

> 用途：把这份文档直接交给 Claude，要求它基于你现有的实验室管理系统进行设计与开发。  
> 目标：实现一个类似牛客网风格的实验室笔试模块，包含管理员出题、学生申请、审批参加、在线考试、防作弊、签名确认、自动/人工阅卷、成绩管理、面试通知等完整流程。  
> 特别要求：前端页面必须现代、美观、清晰，不能是普通后台拼凑风格。

---

## 一、项目背景

当前系统是“高校实验室招新与治理平台 / 实验室管理系统”，已经具备实验室、学生、管理员、报名申请等基础模块。

现在需要在现有系统上扩展一个 **在线笔试中心**，用于各实验室在招新阶段进行线上笔试。

整体业务目标如下：

1. 管理员在后台创建并管理笔试。
2. 管理员可以录入单选题、填空题、编程题等题目。
3. 学生端可以看到各个实验室的笔试申请入口。
4. 学生 **必须先申请某实验室，并且申请状态为“已通过”**，才允许参加该实验室对应的笔试。
5. 笔试过程需要有 **防作弊机制**。
6. 学生开始笔试前需要进行 **诚信承诺签名 / 确认**。
7. 提交后，系统支持自动判分 + 管理员人工批改。
8. 管理员可以查看成绩、排名、到场率、提交率、答题时长等数据。
9. 对于成绩达到要求的学生，管理员可以向学生端发起 **面试邀请 / 面试申请通知**。
10. 前端界面要尽量接近专业招聘考试平台风格，视觉统一，交互顺畅，不能简陋。

## 三、核心业务流程

## 3.1 管理员侧流程

### 1）创建笔试
管理员可以为某个实验室创建一场笔试，配置：
- 笔试名称
- 所属实验室
- 考试说明
- 开始时间
- 结束时间
- 考试时长（分钟）
- 是否允许多次作答（默认否）
- 题目总分
- 是否开启防作弊
- 是否开启签名确认
- 可参加对象（默认：该实验室申请审核通过的学生）

### 2）题目管理
管理员可维护题库，题型至少包括：
- 单选题
- 多选题（可选）
- 判断题（可选）
- 填空题
- 简答题（可选）
- 编程题

每道题支持字段：
- 题目标题
- 题目内容（支持富文本 / Markdown）
- 题型
- 分值
- 难度
- 所属实验室 / 公共题库
- 标准答案
- 解析
- 编程题测试用例
- 编程题支持语言
- 排序号

### 3）组卷
管理员可从题库中手动组卷，后续也可扩展随机组卷。

当前版本优先做：
- 手动选题组卷
- 设置各题分值
- 设置题目顺序
- 预览试卷

### 4）阅卷与成绩管理
管理员可以：
- 查看参考人数、缺考人数、提交人数
- 查看每个学生得分、答题时长、提交时间
- 查看客观题自动评分结果
- 对主观题 / 编程题进行人工批改
- 最终发布成绩
- 按是否通过进行筛选
- 批量向通过者发送面试通知

---

## 3.2 学生侧流程

### 1）查看可申请的实验室笔试
学生端新增一个“笔试中心”或“招新笔试”模块，展示：
- 当前开放招新的实验室
- 实验室简介
- 当前是否已申请
- 审批状态
- 是否可参加笔试
- 笔试时间 / 时长 / 状态

### 2）参加资格限制
必须满足以下条件才可进入考试：
- 当前学生已申请该实验室
- 该申请状态为“审核通过”
- 当前处于考试开放时间内
- 未超过作答次数限制

若不满足条件，页面需明确提示原因，例如：
- 您尚未申请该实验室
- 您的申请尚未审核通过
- 当前考试未开始
- 当前考试已结束
- 您已提交过本场考试

### 3）进入考试前签名确认
若开启签名确认，则学生开始考试前必须完成：
- 阅读诚信考试承诺书
- 勾选“我已阅读并同意”
- 输入姓名签名 / 手写签名（二选一，建议先做输入姓名签名，后续可扩展手写）
- 记录签名时间

签名后方可开始考试。

### 4）在线答题
考试页面需具备类似正规笔试系统的体验：
- 左侧题号导航
- 中间题目区
- 顶部剩余时间倒计时
- 自动保存答题记录
- 支持上一题 / 下一题
- 支持标记未完成
- 提交前二次确认
- 超时自动交卷

### 5）考试结束后
学生可以看到：
- 是否已提交
- 是否等待阅卷
- 已公布成绩时显示分数
- 若通过，显示面试邀请通知

## 四、权限与角色要求

至少包含以下角色：

### 1）超级管理员 / 系统管理员
- 管理所有实验室笔试
- 管理题库

### 2）实验室管理员
- 只能管理自己实验室的笔试
- 只能查看自己实验室的参考学生和成绩
- 只能向自己实验室的通过者发送面试通知

### 3）学生
- 只能查看自己相关实验室的申请状态
- 只能参加自己已通过审批实验室的笔试
- 只能查看自己的考试记录和通知

## 五、防作弊需求（必须实现基础版）

请至少实现一套基础可用的防作弊方案，不要求像商业平台那样复杂，但要体现“已考虑防作弊”。

### 基础版防作弊要求

1. **切屏 / 页面失焦检测**
   - 监听 `visibilitychange`
   - 监听窗口失焦事件
   - 记录切屏次数、时间
   - 达到阈值后进行警告
   - 超过最大阈值可强制交卷或标记异常

2. **禁止复制粘贴（考试页面）**
   - 禁止复制题目文本
   - 编程题可保留编辑区基本输入能力，但限制右键菜单可作为可选项

3. **倒计时结束自动交卷**
   - 防止超时继续作答

4. **定时自动保存**
   - 避免意外刷新导致数据丢失

5. **刷新恢复机制**
   - 学生刷新页面后可恢复到上次答题进度
   - 同时记录刷新行为

6. **单设备会话控制（可选优先）**
   - 同一考生同一考试只允许一个活跃会话
   - 新会话顶掉旧会话或标记异常登录

7. **异常行为日志**
   - 记录切屏、刷新、重复登录、异常提交等行为
   - 供管理员后台查看

### 后续可扩展（本期可预留接口）
- 摄像头抓拍
- 人脸识别
- 屏幕录制
- IP 限制
- 浏览器指纹
