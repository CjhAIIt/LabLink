# 演示脚本

## 推荐时长

8 到 10 分钟。

## 演示顺序

1. `superadmin` 登录
   展示学校总览、学院管理、实验室管理与统计分析页，说明系统解决了实验室信息分散和统计困难问题。

2. 查看实验室详情
   进入实验室管理页，展示实验室简介、指导教师、地点和招新要求。

3. `20231003` 学生登录
   进入学生工作台，查看开放招新计划，演示提交入组申请或查看已存在申请记录。

4. `ai_admin` 登录
   进入申请审核页，审核学生申请，演示从“待审核”到“通过 / 驳回”的流程闭环。

5. 进入成员管理页
   展示成员名册、负责人任命、成员移出等管理能力。

6. 进入公告管理页
   发布一条实验室公告，说明系统兼具内部管理和信息展示能力。

7. 返回统计分析页
   重点讲解：
   - 各学院实验室数量统计
   - 各实验室成员人数统计
   - 招新申请状态分布
   - 月度申请趋势
   - 教师指导实验室数量排行

## 讲解重点

- 主线不是“功能很多”，而是“实验室管理流程被系统化解决”
- 重点强调申请、审核、入组、任命、公告、统计的完整闭环
- 统计分析页用于体现管理信息系统的决策支持价值
