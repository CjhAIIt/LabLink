# 答辩视频录制脚本

## 目标

用于答辩场景，重点不是把系统所有页面点一遍，而是讲清楚项目背景、目标、设计思路、技术实现、创新点和落地价值，并用少量关键操作证明系统真实可用。

## 建议时长

6 到 10 分钟。

## 答辩视频建议结构

1. 项目背景与痛点
2. 项目目标与解决方案
3. 系统架构与技术实现
4. 核心功能与权限设计
5. 关键演示
6. 创新点与应用价值
7. 总结

## 录制前准备

1. 打开答辩 PPT。
2. 浏览器准备好系统页面。
3. 保证 PPT 和系统页面都能快速切换。
4. 如果答辩要求露脸，可用“摄像头小窗 + 屏幕录制”；如果无要求，纯录屏即可。

## 分段脚本

| 时间 | 做什么 | 说什么 |
| --- | --- | --- |
| 0:00 - 0:30 | PPT 首页，展示项目名称、团队、指导老师 | 各位老师好，我们的项目是“安徽信息工程学院实验室管理平台”。本项目面向学校实验室管理实际场景，重点解决实验室建设、成员管理、资料管理和考勤管理中流程分散、权限混乱、统计困难的问题。 |
| 0:30 - 1:20 | 切到“项目背景 / 问题分析”页 | 通过前期调研我们发现，传统实验室管理通常依赖纸质材料、Excel 和微信群通知，存在三个明显问题：第一，信息分散，学校无法统一掌握实验室情况；第二，审批链条不透明，教师建实验室、学生入组都缺少闭环；第三，资料和考勤难以沉淀，后续统计和追溯成本很高。 |
| 1:20 - 2:00 | 切到“项目目标”页 | 因此，我们希望搭建一个覆盖学校、学院、实验室、教师和学生的统一平台，实现多角色分权管理、在线审批协同、资料沉淀归档、晚自习考勤以及统计分析。 |
| 2:00 - 3:00 | 切到“系统架构 / 技术栈”页 | 在技术实现上，项目采用前后端分离架构。前端使用 Vue 和 Element Plus 构建交互界面，后端基于 Spring Boot、Spring Security、MyBatis-Plus 实现业务与权限控制，数据库使用 MySQL 存储业务数据。系统同时支持文件上传、数据导出、邮件验证码等能力。 |
| 3:00 - 3:50 | 切到“核心设计”页，讲权限模型 | 在设计上，我们重点做了权限重构。学校总负责人、学院负责人、指导老师、实验室管理员、学生五类身份权限被重新划分。不同角色的数据范围和操作范围独立控制，例如学校负责人看全校，学院负责人看本学院，指导老师和实验室管理员只看自己所属实验室。 |
| 3:50 - 4:30 | 进入系统，登录 `superadmin`，展示管理端首页 | 接下来我通过系统实际运行页面说明设计已经落地。现在登录的是学校总负责人账号，可以看到进入的是管理端，而不是教师端，说明固定岗位账号和权限范围已经正确绑定。 |
| 4:30 - 5:10 | 展示学院管理、实验室创建审批、工作台范围切换 | 学校总负责人可以统一查看学院管理、实验室创建审批和工作台数据；学院负责人则只能处理本学院任务。这一设计保证了平台既能集中管理，又不会造成越权访问。 |
| 5:10 - 5:50 | 切换学生端，展示加入实验室申请、简历要求、权限隐藏 | 学生端支持实验室浏览和加入申请。系统要求学生申请加入实验室时先提交简历，并且对于未加入实验室的学生，不显示无权限功能入口，从交互层面减少报错和误操作。 |
| 5:50 - 6:30 | 展示考勤任务、名单导出、资料导出 | 除了审批和成员管理，我们还实现了晚自习考勤任务以及名单导出功能，报名名单、考勤名单和实验室资料都可以导出，满足学院和学校的日常管理与归档需求。 |
| 6:30 - 7:20 | 回到 PPT，展示创新点页 | 项目的创新点主要体现在三个方面：一是将学校、学院、实验室多层级治理统一到一个系统；二是围绕真实业务重构权限和范围控制；三是把审批、资料、考勤、导出和统计串成闭环，而不是只做单点页面展示。 |
| 7:20 - 8:00 | 停留总结页收尾 | 总体来看，本项目不仅完成了功能实现，更强调真实管理场景中的可用性和可落地性。系统已经具备清晰的组织结构、稳定的权限模型和完整的业务链条。我的答辩陈述到这里结束，感谢各位老师。 |

## 适合直接照读的“项目亮点”表述

### 亮点一：权限分层清晰

我们没有停留在简单的管理员、教师、学生三角色模型，而是进一步细分为学校总负责人、学院负责人、指导老师、实验室管理员和学生，并给每类身份配置了独立的数据范围和业务权限。

### 亮点二：流程闭环完整

系统把教师注册、实验室创建、学生入组、资料归档、考勤管理和数据导出串联起来，形成从申请到管理再到统计的完整闭环。

### 亮点三：强调实际落地

我们不是只做静态页面，而是面向学校真实使用场景设计功能，例如简历前置校验、无权限功能隐藏、名单导出、资料导出和邮件验证码找回等。

## 答辩时建议重点回答的问题

### 如果老师问“为什么做这个题目”

可以回答：因为学校实验室管理涉及多个层级和角色，现有管理方式往往信息分散、流程不透明，项目具有明确的实际需求和应用价值。

### 如果老师问“你的创新点是什么”

可以回答：创新点主要体现在多层级权限治理、完整业务闭环，以及将实验室资料、考勤和审批统一到一个平台中。

### 如果老师问“难点是什么”

可以回答：难点主要在权限边界划分、跨角色流程联动，以及保证不同身份只能看到对应范围的数据。我们通过统一权限模型和岗位绑定机制解决了这个问题。

### 如果老师问“还能怎么扩展”

可以回答：后续可继续扩展设备借用、项目成果管理、比赛管理、移动端签到、消息通知中心等能力。

## 录制注意事项

1. 答辩视频重点是讲清楚“为什么这样设计”，不是只展示页面。
2. PPT 讲解和系统演示比例建议控制在 6 比 4。
3. 每个模块只展示最关键的一步，避免视频过长。
4. 如果中途切系统页面，提前在浏览器标签页排好顺序。
5. 结束前一定要有一句总结，形成完整答辩闭环。
