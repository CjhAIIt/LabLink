# 作品演示视频录制脚本

## 目标

用于展示“安徽信息工程学院实验室管理平台”已经可以完整支撑学校、学院、实验室、教师、学生多角色协同，重点体现系统能解决什么问题、核心流程如何闭环、实际操作是否顺畅。

## 建议时长

5 到 8 分钟。

## 录制前准备

1. 关闭与比赛无关的软件通知。
2. 浏览器提前登录好需要演示的账号，或将账号密码放在记事本便于切换。
3. 准备好 3 类账号。
   - 学校总负责人：`superadmin`
   - 学院负责人：如 `cs_admin`
   - 学生账号：任选一个已注册学生
4. 浏览器缩放调到 100%，确保侧边栏、表格、按钮都完整可见。
5. 提前准备一页简短开场词，避免录制时卡顿。

## 建议录制方式

- 全程录屏，保留鼠标操作。
- 以真实系统操作为主，不要只放 PPT。
- 每切换一个角色，先说“当前身份是谁，要解决什么事情”，再开始点页面。

## 分段脚本

| 时间 | 做什么 | 说什么 |
| --- | --- | --- |
| 0:00 - 0:20 | 打开系统首页或登录页，停留 2 到 3 秒 | 大家好，我们的作品是“安徽信息工程学院实验室管理平台”，它面向学校、学院、实验室、教师和学生五类使用场景，解决实验室管理分散、审批链条不清、资料沉淀困难和考勤统计低效的问题。 |
| 0:20 - 0:50 | 登录 `superadmin`，进入管理端首页 | 现在首先以学校总负责人身份进入系统。学校总负责人具备全校范围的统筹管理能力，可以统一查看学院、实验室、审批、工作台和统计信息。 |
| 0:50 - 1:30 | 展示学院管理、实验室管理、综合检索或统计页 | 在学校端，可以维护学院主数据、查看实验室总体情况，并对全校实验室建设和运行情况进行统一监管。这一层解决的是过去信息分散、无法从学校视角统筹的问题。 |
| 1:30 - 2:10 | 进入实验室创建审批页面，展示学校审批链路 | 这里展示实验室创建审批流程。系统支持学校与学院两级审批，教师发起申请后，学院先审，学校再审，流程状态透明可追踪，避免线下反复沟通。 |
| 2:10 - 2:40 | 退出并登录 `cs_admin` 或其他学院负责人 | 下面切换到学院负责人身份。学院负责人只看到本学院范围内的数据和任务，权限边界清晰，不会误操作其他学院内容。 |
| 2:40 - 3:20 | 展示学院负责人可见的创建审批、教师注册审批、考勤任务 | 学院负责人可以审核本学院实验室创建申请、教师注册申请，并管理本学院的晚自习考勤任务。这体现了系统按组织层级分权，而不是所有管理都集中在一个账号里。 |
| 3:20 - 4:00 | 进入实验室工作台，切换学院/实验室或查看本学院工作台 | 在工作台模块里，学校总负责人可以跨学院查看，学院负责人查看本学院，指导老师和实验室管理员查看自己负责的实验室。系统把查看范围和操作权限做了严格区分。 |
| 4:00 - 4:40 | 退出并登录学生账号，进入实验室总览、申请页 | 下面切换到学生端。学生可以浏览实验室信息、查看招新计划、提交入组申请，并在个人中心维护简历等资料。 |
| 4:40 - 5:20 | 演示学生加入实验室申请流程，顺带展示未加入实验室时隐藏功能 | 这里可以看到，学生在未加入实验室时，系统不会展示无权限使用的实验室功能；只有在加入后，才会出现考勤、资料空间等入口。这避免了用户误点报错，提升了使用体验。 |
| 5:20 - 6:00 | 返回管理端或实验室工作台，展示资料管理、导出功能 | 系统还支持实验室资料空间管理，以及报名名单、考勤名单、资料文件等导出功能，方便实验室管理员和学院负责人留档、汇总和上报。 |
| 6:00 - 6:30 | 停在系统总览页收尾 | 总结来说，我们的作品不仅实现了实验室信息展示，更形成了“申请、审批、入组、考勤、资料沉淀、统计分析”的完整闭环，具备真实落地使用价值。 |

## 可直接照读的收尾话术

本系统以学校实验室治理场景为核心，将多角色、多层级、多流程统一到一个平台中，实现了职责清晰、权限明确、流程透明、数据可沉淀、结果可统计的目标。以上就是我们的作品演示，谢谢各位老师。

## 录制注意事项

1. 不要长时间停留在空白表格上。
2. 每次点击前先说目的，例如“现在查看学院审批结果”“现在导出报名名单”。
3. 如果某页数据较少，重点讲“为什么这样设计”，不要只说“这里有个表”。
4. 如果需要切账号，提前退出到登录页，避免录制中找按钮找太久。
5. 鼠标移动慢一点，语速稳定，不要抢着点。

## 建议备用台词

- 这一模块解决的是线下管理依赖表格和聊天记录的问题。
- 这一权限设计保证了不同身份只能看到自己应该看到的数据。
- 这一流程体现了学校、学院、实验室三级协同的业务闭环。
- 这一导出能力方便留档、统计和上报，不需要再手工整理名单。
